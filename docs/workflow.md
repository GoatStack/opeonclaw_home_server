# Workflow

## 목적
이 프로젝트는 서버에서 자동화 작업을 수행하고, 노트북에서는 Cursor로 중간 확인만 하는 흐름을 목표로 한다.

## 기본 흐름
1. 사용자가 `tasks/`에 작업 요청을 남긴다.
2. OpenClaw가 작업을 분석하고 필요한 파일/구조를 만든다.
3. 코드와 문서 변경사항은 저장소에 반영한다.
4. 사용자는 노트북에서 Cursor로 검토한다.
5. 필요하면 추가 지시를 내려 다음 작업을 진행한다.

## 작업 단위
작업은 가능한 한 작게 쪼갠다.
예시:
- 저장소 구조 만들기
- README 정리
- 실행 스크립트 추가
- 특정 자동화 모듈 구현
- 테스트 및 로그 추가

## 로그 원칙
- 중요한 작업은 `logs/`에 남긴다.
- 실패 원인과 수정 내역을 간단히 기록한다.
- 반복 작업은 스크립트로 옮긴다.

## Cursor 연동 방식
- Cursor는 노트북에서 저장소를 열어 확인용으로 사용한다.
- 서버가 실제 작업 수행 위치가 된다.
- GitHub가 동기화 허브 역할을 한다.

## 운영 원칙
- 무거운 작업은 서버에서 수행
- 검토는 노트북에서 수행
- 반복되는 일은 문서화 후 스크립트화
